Systemwide Reports

SchoolMessenger contains systemwide reporting features that help you know exactly what your system is doing at any given time including:

  • Active & Pending Broadcasts

  • Completed Broadcasts

  • Repeating Broadcasts

 

Active & Pending Broadcasts

The systemwide Active & Pending Broadcasts screen shown below can be accessed by clicking the System tab followed by the Active Broadcasts tab. This screen shows all of the Broadcasts that are scheduled or currently running for every user. It contains several columns of information.

  • Submitted by: This column shows which user submitted which Broadcasts.

  • Broadcast Name: This column shows the name of the Broadcast that is active or pending.

  • Status: This column shows whether or not the Broadcast has been submitted as well as if it is active or cancelled.

  • Total: This shows the number of people who will be contacted by this Broadcast and what types of communication will be used (phone or email).

  • Remaining: This column shows how many people have not yet been contacted by this Broadcast.

  • Scheduled Start: This column shows when the Broadcast is scheduled to run.

  • Actions: This column contains two links. Clicking Login as this User will log you in as that use so that you may edit the Broadcast and Cancel will cancel the Broadcast so that it stops running or does not run at all.

 

Completed Broadcasts

The systemwide Completed Broadcasts screen shown below displays all of the Broadcasts that the system has completed for every user. It can be accessed by clicking on the System tab followed by the Completed Broadcasts tab. It contains several columns of information explained below.

  • Submitted by: This column shows which user submitted which Broadcast.

  • Broadcast Name: This column shows the name of the Broadcast that was completed.

  • Start Date: This column shows the date when the Broadcast ran.

  • Status: This column shows if the Broadcast was complete,

  • Total: This column shows the number of people contacted and by what method.

  • Success Rate: The Success Rate shows how many messages were successfully delivered (ie. phone calls were answered by a person or machine versus no answer at all).

  • Scheduled End Date/Time: This column shows when the Broadcast was scheduled to stop running whether or not it was finished.

  • Actions: This column contains three links. Login as this User lets you log into the user account which created the Broadcast, report lets you see more details about the Broadcast, and archive which allows you to archive the Broadcast, causing it to no longer display on the user's Completed Broadcasts page.

 

Repeating Broadcasts

The systemwide Repeating Broadcasts screen shown below displays all of the repeating Broadcasts currently scheduled in the system. It can be accessed by clicking the System tab followed by the Repeating Broadcasts tab. It contains several columns of information explained below.

  • Submitted by: This column shows which user submitted which Broadcasts.

  • Broadcast Name: This column shows the name of the Broadcast that was completed.

  • Next Scheduled Run: This column shows the next time that the Broadcast is scheduled to run.

  • Actions: The column contains two links. The first, Login as this user (or Edit if it is your own Broadcast) allows you to make changes to the Broadcast. The second, Delete, removes the Broadcast from the system.
