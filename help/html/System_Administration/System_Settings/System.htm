Systemwide Alert Message:

Systemwide Alert Message - Text entered here will appear as an alert message for all users in the system. The alert message appears as black text in a red box on every page.

 

Customer Information:

Customer Display Name - Enter the name that you would like to appear at the top of each page.

Email Domain - Entering an email domain here will prevent users from sending emails with a different "From" address.

Local Area Code - Enter the local area code for your location.

Field Definitions:

Important: Modifying data field definitions after users have begun using your system can cause unexpected results. Please contact support services for assistance before modifying your field definitions. Call 800-920-3897.

By default, your system will contain First Name, Last Name, and Language metadata as shown in the image above. These fields cannot be deleted or edited in any way.

There are four parts to the form that you will need to consider when creating a new field. They are:

Field Number - This is the field in the database that your data will occupy.

Name - This is the name that represents the data in the field. It is what users will see when they see this field in menus throughout the system.

Type - There are three types you can assign to your new field. These correspond to how and where the field will appear in the system. All of them are potentially available for creating dynamic messages.

List - List will list all possible values for the data field in a drop down menu. This is good for fields such as school name which will have a limited number of values. Fields with the List type are used to create rules throughout the system such as when creating lists, performing searches, or restricting users to certain Data Views.

Text - Text will give users the ability to search the data by typing into a text box. This is good for fields which will have many possible values such as first and last name. Fields with the text type are only used on search pages such as the Contact Database and List Search.

Date - Defines data that contains a date, such as Absence Date.

Searchable - Checking this box determines whether or not users will be able to search on this field. Not all fields should be searchable. This should be checked for fields like Grade which have a set number of possible values.

Group Field Definitions:

Group Field definitions allow you to specify up to ten fields of additional data for your contacts. For example, if your data contains afterschool activities, you can create an Afterschool group field. All group fields are automatically created as searchable fields so they can be used to create lists.

 

Enrollment Field Definitions:

Enrollment Fields are used to associate students with staff and any additional class information your data contains. This allows users to create lists based solely on class enrollment, so for example, a teacher can easily make a list of his students based on the class the student is enrolled in.

 

Security:

Minimum Username Length - Enter the minimum number of characters that a username may contain.

Minimum Password Length - Enter the minimum number of characters that a password may contain.

Invalid Login Lockout - Enter the number of failed login attempts permissible before the system should temporarily lock the account.

Invalid Login Lockout Period - Enter the number of minutes an account should be locked out when there have been invalid login attempts.

Invalid Login Disable Account - Enter the number of invalid login attempts permissible before the system should disable the account. This must be greater than the number of invalid logins permissible before a temporary lockout.

 

Display Settings:

 

Color Theme - Use the dropdown menu to select a theme for the system's default user interface.

Primary Color (in hex) - Enter the value of the color you would like for the text and shaded areas throughout the system. This needs to be the hex value for the color like you would use in HTML. You may also click on the palette icon to the right of the field to select a color from the menu.

Ratio of Primary to Background - This setting controls how much of your primary color is blended with the shaded areas in the interface. An example shaded area can be seen on the left side of the image above.
