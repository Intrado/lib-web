Managing User Accounts

Important: Only those features that you have been granted access to will display. For access to additional features contact your system administrator.

Users are people who have access to the system either using a web browser, a telephone, or both. The system administrator can create and configure new user accounts from the User List, found under the Admin tab in the Users subtab. The image below shows the User List screen which will be explained below.

Active Users:

This section displays all of the users currently entered into the system and the Access Profile they are currently set at. It also contains the Add New User button which you must click to add a new user. To the right in the Actions area are four links:

  • Edit - Clicking this takes you to the User Editor where you can configure the user's account.

Note: If the user is part of a user import, only the password and PIN will be editable.

  • Login as this user - Allows you to log into the system as that user.

  • Disable - Clicking this link renders that user inactive. They will be displayed in the Inactive Users section of the page.

 

Inactive Users:

This section displays the users whose accounts have been disabled. To the right in the Actions area there are three links:

  • Edit - Clicking this takes you to the User Editor where you can configure the user's account.

  • Enable - Allows the user to login and moves the user's account back into the Active Users section of the page.

  • Delete - Clicking this link deletes the user from the system. Deleting cannot be undone.

 

The User Editor Screen:

The User Editor screen is where user's accounts can be configured. The image below shows the User Editor screen and information on each section can be found below.

Note: If the user is part of a user import, only the password and PIN will be editable.

Access Credentials:

  • Enter the users information into the fields provided. It is important to remember that the user name and password are case-sensitive. The Telephone User ID and Pin Code must be numeric. The Email field is for the user's email address and is where the link to the password reset feature will be sent if the user forgets or needs to change their password. Auto Report Emails are the additional email addresses where reports should be sent after a job is completed. Multiple email addresses in this field may be separated by commas or semicolons.

 

Restrictions:

  • Access Profile - Choose an Access Profile from the drop down menu

  • Restrict this user to the following types of jobs - Check this and select the job types this user should be able to do. Hold down the Ctrl key (Command key on Macs) to select multiple choices.

  • Restrict this user to the following types of survey jobs - Check this and select the types of survey jobs this user should be able to do. Hold down the Ctrl key (Command key on Macs) to select multiple choices.

 

Data View:

  • User's data access can be restricted by Data or Staff ID.

By Data - Select this to restrict a user to certain types of data. For example, if the user should only be able to call any student in Anytown Elementary, you could select School Name is Anytown Elementary, which would restrict them to contacts who are associated with Anytown Elementary.

By Staff ID - Select this to restrict a user to only contacts associated with the user's Staff ID. Staff ID associations are managed by the Enrollment Import data. For example, if the user was a teacher who only needed to be able to communicate with students enrolled in his classes, selecting this option would only allow him to see students with his Staff ID in their enrollment data.  

Email to Reset Password:

  • Click this button to send an email to the user which guides them through resetting their password. They will need to click a link in the email which will take them to a screen where they may enter a new password.

When you have finished making changes, click the Save button to save your changes.
