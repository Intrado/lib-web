Creating an Advanced Phone Message

Important: Only those features that you have been granted access to will display. For access to additional features contact your system administrator.

Phone messages can be simple (an unchanging message with a basic phone recording) or advanced (using advanced options or dynamic data fields). The following sections will outline the methods you can use to create an advanced phone message. If you need to learn more about creating a simple message please read the help topic, Creating a Simple Phone Message.

The most common use for an advanced message is to personalize the content with dynamic data elements, however not all advanced messages are dynamic. For example you could just type your phone message into the Phone Message Editor and have a text-to-speech voice read it to each recipient.

To create an advanced phone message with dynamic data elements it helps to have a basic understanding of what a message is composed of. An advanced message contains both static and dynamic elements which combine to form a cohesive personalized message. The static elements can include audio files or text that has been typed into the Phone Message Editor. The dynamic elements are the data fields you can insert using the Phone Message Editor. The data fields represent fields from your student database and the data will be spoken by your chosen text-to-speech voice.

When recording audio files for your dynamic message, it's important to remember to record the static portions of your message in individual files to allow for the insertion of data fields when constructing the final message.

For example:

Desired End Result:

"Hello. This is Oakdale Elementary calling to inform you that your child, <Johnny Smith>, was marked absent today."

This message would be recorded in two parts:

Part One: "Hello. This is Oakdale Elementary calling to inform you that your child,"

Part Two: "was marked absent today."

Once the recordings are entered into the Audio File Library, the message can be assembled in the Phone Message Editor screen.

There are three different ways to get the static portion of your dynamic phone message into the system. The following sections will discuss each method.

Recording a Message Using the Telephone:

The simplest way to record an audio file for use in your dynamic message is using a telephone with the Call Me to Record feature in the Audio File Library.

Clicking on the Call Me to Record link at the top of the Audio File Library will take you to the Call Me to Record screen shown here.

  1. Click the Call Me to Record button on the Message Builder page.

  2. Enter a name for your new message and the direct access phone number where you can be reached.

  3. Click Call Me to Record.

  4. You will receive a call from the system momentarily.

  5. Follow the prompts to record your message. To record more than one message, simply follow the prompts.

  6. When you are done, your new messages will appear in the Call Me window, as shown below, where you can assign them names.

  7. Click Save when you are done to save your messages and exit Call Me to Record.

Recording a Message Using Your Computer:

In addition to recording a message over the phone, you can also record a message using any other program capable of recording a standard audio file (.wav, .aiff, or .au). The audio file can then be uploaded for use in your message by using the Upload Audio File feature in the Audio File Library. The same considerations applied to recording a message with the telephone apply to messages recorded on your computer. Dynamic messages must be uploaded in multiple parts to allow for the insertion of data fields using the Phone Message Editor as in the example outlined above.

Creating a Text-to-Speech Phone Message:

A telephone message can be created entirely from text by simply typing the message and allowing the system to convert the message into a spoken voice using the built-in text-to-speech conversion engine. To create a text-to-speech message, start by creating a new phone message in the Message Builder screen, and then simply type the text of your message into the Message section of the Phone Message Editor window. You may also insert data fields to create dynamic messages as well. Click the Save button to save your message and return to the main Message Builder page.

Note: When creating multilingual messages it's important that you select the correct text-to-speech voice for the language being used. An English text-to-speech voice cannot translate Spanish and vice versa. Currently English and Spanish, male and female voices are available.

The Phone Message Editor:

Click the Create Advanced Phone Message button to assemble a new phone message. You will be taken to the Phone Message Editor screen where you can enter a brief name and description for your message. You can then assemble your message from your audio files and insert any data fields you would like to include. If you plan to create this message in an alternate language be sure to indicate the language in the message name for easy identification when you create your notification job.

Assembling Your Message Content:

Important: Do not edit the bracketed text which appears when you insert audio files or data fields. Doing so may cause your message to not function properly.

For a Multi-Part Message with Dynamic Content:

  1. Select the audio file for the first part from the Audio Recording drop down menu on the right side.

  2. Click the Play button to preview the audio file.

  3. Click the Insert button to place the file in your message.

  4. Select the Data Field you wish to insert from the Data Fields drop down menu on the right side and click the Insert button. You may wish to include a value in the Default Value space so that the system will have something to say in the event that there is no data in your database for the specified data field for a particular student.

  5. Select the next audio file for the message and repeat steps 1 through 4 until your message is complete.

  6. Select which text-to-speech voice you prefer for your dynamic content.

  7. Click the Save button at the top or bottom of the screen when your message is complete. You may preview your message with sample data values by clicking the Play button next to your message on the Message Builder screen.

Note: When creating multilingual messages it's important that you select the correct text-to-speech voice for the language being used. An English text-to-speech voice cannot translate Spanish and vice versa. Currently English and Spanish, male and female voices are available.

Formatting a Multilingual Message Using Text-to-Speech:

When creating a multilingual text-to-speech message, it is important to choose the correct voice for each part of the message. Text-to-speech voices are designed to correctly read words written in the target language and often will not do well when attempting to read words from another language. Adding some simple tags to your message will allow the correct voice to be selected when rendering your message. For example, if you are creating a message in Spanish, you may wish to select the English voice for the school name because phrases like "Elementary School" will not be correctly rendered by the Spanish voice.

Simply select the main voice for your message from the drop down menu on the right side of the Message Editor. Then when you would like to switch to a different language for a few words, just put the language in double square brackets. When you wish to return to the original language, just insert it in square brackets at the point where it should begin to be used again. Gender will be maintained throughout, so if the main voice that you selected was "Spanish-Female" then any other languages will also be female.

Here is an example message written in Spanish and English:

Hola.  Este es un mensaje de [[English]] Anytown High School [[Spanish]] informandole que su hijo o hija,fue marcado ausente hoy. Por favor contacte [[English]] Anytown High School [[Spanish]]  al (345) 123-4560 para aclarar esta ausencia, o envie una nota firmada explicando la razon por esta ausencia cuando su hijo o hija regrese a la escuela.  Si ya ha contactado la oficina de la escuela, no tome en cuenta este mensaje.  Gracias.

With this formatting, the words "Anytown High School" will be in English and then Spanish will resume right after [[Spanish]]. It is not necessary to put [[Spanish]] at the beginning of the message because the voice selected from the menu was Spanish, therefore Spanish is the default language for this message.
