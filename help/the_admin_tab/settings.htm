Settings

Important: Only those features that you have been granted access to will display. For access to additional features contact your system administrator.

The Settings subtab contains options for the system, job, and destination settings. This subtab also allows you to manage Contact Manager Activation tokens and set the information that Contact Manager users will see, if you have purchased the Contact Manager option.

System:

The links in this section allow you to modify settings for the entire system.

Systemwide Alert Message:

Systemwide Alert Message - Text entered here will appear as an alert message for all users in the system. The alert message appears as black text in a red box on every page.

 

Customer Information:

Customer Display Name - Enter the name that you would like to appear at the top of each page.

Email Domain - Entering an email domain here will prevent users from sending emails with a different "From" address.

Local Area Code - Enter the local area code for your location.

Field Definitions:

Important: Modifying data field definitions after users have begun using your system can cause unexpected results. Please contact support services for assistance before modifying your field definitions. Call 800-920-3897.

By default, your system will contain First Name, Last Name, and Language metadata as shown in the image above. These fields cannot be deleted or edited in any way.

There are four parts to the form that you will need to consider when creating a new field. They are:

Field Number - This is the field in the database that your data will occupy.

Name - This is the name that represents the data in the field. It is what users will see when they see this field in menus throughout the system.

Type - There are three types you can assign to your new field. These correspond to how and where the field will appear in the system. All of them are potentially available for creating dynamic messages.

List - List will list all possible values for the data field in a drop down menu. This is good for fields such as school name which will have a limited number of values. Fields with the List type are used to create rules throughout the system such as when creating lists, performing searches, or restricting users to certain Data Views.

Text - Text will give users the ability to search the data by typing into a text box. This is good for fields which will have many possible values such as first and last name. Fields with the text type are only used on search pages such as the Contact Database and List Search.

Date - Defines data that contains a date, such as Absence Date.

Searchable - Checking this box determines whether or not users will be able to search on this field. Not all fields should be searchable. This should be checked for fields like Grade or Afterschool Program which have a set number of possible values.

 

Security:

Minimum Username Length - Enter the minimum number of characters that a username may contain.

Minimum Password Length - Enter the minimum number of characters that a password may contain.

Invalid Login Lockout - Enter the number of failed login attempts permissible before the system should temporarily lock the account.

Invalid Login Lockout Period - Enter the number of minutes an account should be locked out when there have been invalid login attempts.

Invalid Login Disable Account - Enter the number of invalid login attempts permissible before the system should disable the account. This must be greater than the number of invalid logins permissible before a temporary lockout.

 

Display Settings:

 

Color Theme - Use the dropdown menu to select a theme for the system's default user interface.

Primary Color (in hex) - Enter the value of the color you would like for the text and shaded areas throughout the system. This needs to be the hex value for the color like you would use in HTML. You may also click on the palette icon to the right of the field to select a color from the menu.

Ratio of Primary to Background - This setting controls how much of your primary color is blended with the shaded areas in the interface. An example shaded area can be seen on the left side of the image above.

 

Job:

The links in this section allow you to edit settings for all jobs in the system.

Disable Repeating Jobs:

Disable Repeating Jobs - Click this checkbox to prevent the system from performing any of its scheduled repeating jobs. Unchecking this box will allow the repeating jobs to resume at their next scheduled time.

 

Job Settings:

Retry Settings - Use the drop down menu to select the amount of time the system should wait in between attempts to call an unanswered phone number.

Default Caller ID Number - Enter the phone number you would like to display in the recipient's Caller ID.

Autoreport Email Address - The email address entered in this field will be displayed as the from address users will see in the reports which are emailed to them. It is important to set this so that the users will see that the emailed report is from the district and so that spam filters will not block the report emails.

Autoreport Email Name - The name entered here will be the name that reports are sent from. For example, "Anytown School District".

 

Job Types:

Name - This is the name of the job type in the system. It will be seen by users when creating or viewing jobs in the system. It will also be seen by Contact Manager users when they view the types of jobs that they can receive.

Note: The Emergency job type may not be edited.

Display Information - Enter a description for each job type which will be viewed by users. It will also be viewable by Contact Manager users, if this feature is enabled on your account.

Default Notification Setting - Check which contact methods the job type will use by default when contacting message recipients. When you mouse over each checkbox, the full name associated with each contact method will be displayed.

Delete - Clicking this button will delete the job type from the system. Emergency and High Priority job types may not be deleted.

 

Create New Job Type:

You may create a new job type by clicking on the Create New Job Type button in the Job Type screen.

Note: New job types may only be General priority.

Name - Enter the name of the Job Type. This is how it will be seen by users creating jobs as well as by portal users.

Check if this is a Survey - Check this box if this job type is for surveys.

Display Information - Enter the description of the job type as you would like it to be seen by portal users.

Default Contact Preferences - Check the default methods that the job will use to contact recipients.

 

Destination Labels:

The links in this section allow you to name the different destination types in the system, for example, Phone 1 could be set to "Home Phone". The information entered in these pages will be displayed throughout the system.

Email Labels:

Clicking this link will take you to the page for editing Email labels, shown below. Choose one of the system default labels from the drop down menu, or select Other and enter a custom label. Click the Done button when you have finished creating labels.

SMS Labels:

Clicking this link will take you to the page for editing SMS labels, shown below. Choose one of the system default labels from the drop down menu, or select Other and enter a custom label. Click the Done button when you have finished creating labels.
