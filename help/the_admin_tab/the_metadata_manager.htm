The Metadata Manager

The Metadata Manager allows you to configure custom fields in your system's database to be usable for lists, searches, user profiles, and dynamic messages. You may have a total of 20 different metadata fields, however three of these fields are occupied by first name, last name, and language. You can get to this screen by clicking on the Admin tab followed by the Metadata tab. After defining the fields in the Metadata Manager, you will need to create a data import in the Import Manager in order to populate the data.  

There are three types of data in the system, two of which are available in the system by default and should not be mapped here. The three types of data are:

Standard Data - This data includes Unique ID and destination data like phone numbers, addresses, emails, etc. Fields of this type should not be mapped here as they are available by default in the system.

User Data - This data includes user account related data such as user logins. Fields of this type should not be mapped here as they are available by default.

Metadata - This data includes information such as lunch balance, absent date, and school name. Metadata can be any custom data that you have available for export from your database that is not covered by one of the default data types. You can create new fields for this data in the system's database using the Metadata Manager and then use the Task Manager to map the data to those fields.

There are four parts to the form that you will need to consider when creating a new field. They are:

  1. Field Number - This is the field in the database that your data will occupy.

  2. Name - This is the name that represents the data in the field. It is what users will see when they see this field in menus throughout the system.

  3. Type - There are three types you can assign to your new field. These correspond to how and where the field will appear in the system. All of them are potentially available for creating dynamic messages (depending upon the settings in the user's Access Profile).

List - List will list all possible values for the data field in a drop down menu. This is good for fields such as school name which will have a limited number of values. Fields with the List type are used to create rules throughout the system such as when creating lists, performing searches, or restricting users to certain Data Views.

Text - Text will give users the ability to search the data by typing into a text box. This is good for fields which will have many possible values such as first and last name. Fields with the text type are only used on search pages such as the Contact Database and List Search.

Date - Date will create fields that fill with a date value of today, yesterday, or last weekday. It is used primarily for fields like Absent Date.

  1. Searchable - Checking this box determines whether or not users will be able to search on this field. Not all fields should be searchable. For example, a field for lunch balance would be impractical to search because users would have to know the specific value they were searching for such as $13.02.
