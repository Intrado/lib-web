The Import Manager

The Import Manager is a powerful tool which allows you to configure new data imports for the system. This page can be accessed by clicking the Admin tab followed by the Imports tab.

Before you configure a new data import, you should assign fields for all of the custom metadata you plan to include using the Metadata Manager.

Creating a New Data Import

There are three basic steps to setting up a new data import in the system.

  1. Create a CSV (Comma Separated Value) file of the data to be imported and upload the file to the system's server.

  2. Add a new import to the Import Manager and schedule it in the Import Editor.

  3. Edit the field mapping on the Data Import Mapping screen.

Each of these steps contain several configuration options that you will need to use in order to successfully import your data. The following sections will explain these options, screen by screen, in detail.

The Import Manager Screen

The Import Manager screen shown in the image below, contains a list of data imports that are currently saved in the system. It also serves as the starting point for adding new data imports to the system.

The various parts of the Import Manager screen are as follows:

Add New Import - Click this button to begin adding a new import to the system. You will be taken to the Import Editor screen where you can configure the new import.

Name - This is the name that was assigned to the import in the Import Editor.

Description - This is the description that was assigned to the import in the Import Editor.

Type - This is the type of data import that has been assigned in the Import Editor.

Next Scheduled Run - If the import was set up to run at a regularly scheduled time, the next time it is scheduled to run will display here.

Status - This shows whether the import is running, idle, or if there was an error.

File - This shows whether or not the system is able to locate the import's CSV file. It's possible values are Exists, Not Found, or Error.

Run Now - Click Run Now if you need to run an import immediately, regardless of its next scheduled run.

Edit - This will take you to the Import Editor screen where you can rename or reschedule the import.

Edit Mapping - This link will take you to the Data Import Mapping screen where you can assign fields to the data you are importing.

Delete - This will delete the import from the system.

 

The Import Editor Screen

The Import Editor screen, shown in the image below, is used to configure data imports.

There are a few parts to the Import Editor. The first part is the Settings area which contains:

Name - Enter the name for the import here.

Description - Enter a description of the type of data being imported here.

File - Enter the complete path to the CSV file on your server here.

Update Method - There are three possible values in this menu.

Full - Adds AND Deletes -This will completely synchronize the database with the data in your import. If there are people in the system's database who are not in the import's CSV file when the import is run again, they will be deleted from the database. If there are new people in the most recent import they will be added to the system's database.

Update - Adds ONLY - This will add any new people from your import to the system's database. If people are not in your import, but exist in the system's database, they will not be deleted.

Update Only - Neither Adds NOR Deletes - This will update existing information for people already in the system's database. If there are new people in the import or people missing from the import, they will not be added or removed from the system's database. Nothing new is created and only existing information is updated.

The next part is the Scheduling area. In this area you can select which days and time you would like the import to run.

 

The Data Import Mapping Screen

After creating and configuring your data import, you must click on the "Edit Mapping" link on the Import Manager. On the Data Import Mapping page, shown below, you will see some sample entries from your CSV file with a drop down menu over each type. Select the data type from the drop down menu to determine which fields the data will be found in.

 

Important Considerations When Mapping Your Data

Unique ID - Whenever a person is associated with a Unique ID, the system will view that person as a potential recipient. When importing students, use the student ID as the Unique ID. If there are members of the staff that need to be recipients, their staff IDs, if available, should be used as their Unique IDs.

Importing Users - Whenever data is labeled with the User Login data type, the system will create a disabled user account with a password that is set the same as the User Login. The new account will have the first and last name, first phone number, and first email address also associated with it if they are mapped as such. When the users are imported, the system will automatically create an Access Profile with all options disabled. Just make sure to properly configure the Access Profile and then the accounts are ready to be enabled.

Importing Users Who Are Also Recipients - You can import users and set them up to be recipients at the same time by assigning them a Unique ID when they are imported. Just follow the directions in the Importing Users section above and additionally label their staff ID, if available, as their Unique ID.
